2-1. 상단 바

※ 1.12.0 버전 스팩에서 제외 됨

  • 보유 중인 뽑기와 관련된 재화를 상단에 출력
  1. 재화 아이콘
    • 각 재화의 아이콘 출력
  1. 재화 보유량
    • 각 재화 보유량 출력
  1. 구매,교환 버튼
    • 선택 시 각 재화 주요 획득처 이동
      • 보석 = 상점 페이지 보석탭 이동
      • 길드코인 = 길드코인 교환 팝업 출력
      • 상시 뽑기권 = 상시뽑기 구매 팝업 출력
      • 한정 뽑기권 = 한정뽑기 구매 팝업 출력

2-2. 뽑기 리스트

  • 현재 활성화 된 뽑기 목록 출력
  • 활성화 조건
    • ProductInfo 테이블 참조
      1. Id == Id 앞 2자리 20
      1. CanBuy == True
      1. Duration${} = Start < 현재 시각 > End
  • 정렬 순서
    • ProductInfo → Duration의 End 날짜 최근 순서대로 내림차순으로 정렬
    • Duration이 공백인 경우 최하단에 배치
  • 잔여시간 출력
    1. 잔여 기간 타이머
      • 뽑기 잔여 기간 출력
        • 잔여기간 = 뽑기 이벤트 종료 시각 - 현재 시각(UTC)
        • 뽑기 이벤트 종료 시각 = 뽑기 상품 Id 참조 ProductInfo → Duration${End}
        • 잔여 기간 24시간 초과 = n 일 남음
        • 잔여 기간 24시간 이하 = n 시간 남음
        • 잔여 기간 1시간 이하 = n분 m초 남음
  1. 뽑기 아이콘 배너
    • 각 뽑기의 배너 이미지 출력

    ※ 뽑기 배너 이미지는 지정 경로에 GachaRewardInfo → Id 로 구분하여 추가한다.

    • e.g) GachaRewardInfo_Banner_300001.png

  1. 현재 탭 표시
    • 현재 선택된 뽑기를 아이콘 뒤에 하이라이터로 표시해준다.

2-3. 뽑기 상세정보

뽑기 기본 정보

  1. 뽑기 타입 정보
    1. 뽑기 타입에 따른 정보를 출력한다.
    1. ProductInfo → Id를 참조하여 출력
  1. 뽑기 확률 버튼
    1. 선택 시 ShopRandomItemDetailPopup 프리팹 출력

      ※ Hive 모듈 사용으로 별도 웹페이지 출력 시 변경 될 수 있음

  1. 뽑기 제목
    1. 뽑기 제목
      • Data.ProductInfo.뽑기 Id.Name,"뽑기 이름"
    1. 안내 문구
      • 고정 문구 출력

  1. 천장 정보
    • 출력 방식 : 뽑기 횟수/천장 한도 [천장 아이템 아이콘]
    • 천장 직전 연출 : 뽑기 횟수 -1 = 천장 한도
      1. 뽑기 횟수 : 색상 변경
      1. 추가 툴팁 출력
    1. 뽑기 횟수 = 해당 뽑기 시도 횟수
      • 각 뽑기마다 횟수 저장 됨
      • 천장 시스템 정책에 따라 뽑기에서 천장 아이템 획득 시 0으로 초기화
    1. 천장 한도 = GachaRewardInfo → PittyCounts 출력
      • GachaRewardInfo 의 Id = 해당 뽑기 ProductInfo → GachaRewardId 참조
    1. 천장 아이템 아이콘 = 하단 경로 참조
      1. GachaRewardInfo → PittyItem 의 Id를 RandomItemSetInfo 참조하여 아이콘 출력 형태 결정
    1. 툴팁
      • 천장 직전에서만 출력되는 툴팁
      • 뽑기 타입에 따라 출력 내용 변경
        1. 상시 뽑기 = "다음 뽑기에서 전설급 아이템 확정 획득"
        1. 한정 뽑기 = "다음 뽑기에서 획득"
  1. 뽑기 버튼
    • 버튼 선택 시 2. 확인 팝업 출력
    • 아래 모든 팝업 출력 시 상단 바가 출력되어야 함
    1. 뽑기 버튼
      • 1회 뽑기 버튼 = 해당 뽑기 1회 진행
      • 10회 뽑기 버튼 = 해당 뽑기 10회 진행
      • 뽑기 버튼 선택 시 2. 확인 팝업 출력

        ※ 뽑기권이 부족할 경우 3. 뽑기권 교환 팝업 출력

      • 텍스트 = n회 뽑기
      • 필요 자원 아이콘 = 해당 뽑기 ID ProductInfo → Payment 타입 참조
        • GachaTicket = 일반 뽑기권 아이콘
        • LimitedGachaTicket = 한정 뽑기권 아이콘
      • 자원 요구량 = n개
        • n = 자원 * 버튼 당 뽑기 횟수
        • 부족 시 텍스트 색상 빨간색으로 출력
    1. 확인 팝업
      • n = 뽑기 횟수
      • m = 소비 자원 량
      • 타이틀 = [뽑기명] n회
      • 설명 = m 개의 [소비 자원 명] 을 사용하여 뽑기를 진행합니다. 계속하시겠습니까?
      • 버튼
        1. 취소 버튼 : 선택 시 팝업 닫기
        1. 확인 버튼 : 선택 시 팝업 닫기 후 뽑기 결과 이동
    1. 뽑기권 교환 팝업
      • n = 보유한 뽑기권 개수 - 필요한 뽑기권 개수
      • m = [뽑기권 가격] * n
        • 뽑기권 가격 = 뽑기권 상품 Id 참조 ProductInfo → Price
      • 타이틀 = [뽑기 자원] 교환
      • 설명 = [뽑기 자원]이 n 개 부족합니다. m개의 [길드 코인 명]을 사용하여 구매 하시겠습니까?
      • 버튼
        1. 취소 버튼 : 선택 시 팝업 닫기
        1. 확인 버튼 : 선택 시 팝업 닫기 후 뽑기권 n개 구매

          ※ 코인이 부족할 시 4. 코인 교환 팝업 출력

    1. 코인 교환 팝업
      • n = [길드코인 보유량] - [뽑기권 가격] * [3. 뽑기권 교환 팝업에서 구매할 뽑기권 개수]
      • 타이틀 = 길드코인 구매
      • 설명 = 길드코인이 n 개 부족합니다. 보석n개를 사용해 동일한 양의 길드코인으로 교환하시겠습니까?
      • 버튼
        1. 취소 버튼 : 선택 시 팝업 닫기
        1. 확인 버튼 : 팝업 닫기 후 선택 시 길드코인 n개 교환

          ※ 보석이 부족할 시 5. 보석 구매 팝업 출력

      1. 보석 교환 팝업
      • 타이틀 = 보석 구매
      • 설명 = 보석이 부족합니다. 보석 구매 화면으로 이동하시겠습니까?
      • 버튼
        1. 취소 버튼 : 선택 시 팝업 닫기
        1. 확인 버튼 : 선택 시 팝업 닫기 후 상점의 보석 구매 화면으로 이동

한정 뽑기 추가 정보

  • 한정 뽑기 타입에서 추가되는 정보
  1. 한정 캐릭터 정보

    • 해당 뽑기에서 천장 대상인 캐릭터 정보 출력
    • 출력 내용 : 기간 중 '[천장 캐릭터 명]' 출현 확률 UP

4. 상점

  • 상점 리스트 출력 순서
    1. 보석 구매
    1. 길드코인 교환
    1. 뽑기권 교환
    1. 골드 구매
  1. 길드코인 교환 버튼
    • 선택 시 길드코인 교환 팝업 출력
    1. 길드코인 아이콘
    1. 길드코인 비용 = 보석 아이콘
  1. 뽑기권 교환 버튼
    • 선택 시 각 뽑기권 구매 팝업 출력
    1. 뽑기권 아이콘
      1. 뽑기권 Id 참조 ProductInfo → FixedRewardId 참조
      1. FixedRewardInfo → Item Id로 해당 아이템 아이콘 호출
    1. 뽑기권 비용
      1. 아이콘 = 뽑기권 Id 참조 ProductInfo → Payment 참조
      1. 값 = 뽑기권 Id 참조 ProductInfo → Price 참조

4-1. 길드코인 교환 팝업

※ 1.12.0 버전 스팩에서 제외 됨

  • 길드코인 교환 버튼 선택 시 출력
  • 길드코인은 보석과 1대1 교환
  • 해당 팝업 출력 시 상단 바가 출력되어야 함
  1. 타이틀 = 길드코인 교환
  1. 교환비 정보
    1. 보석
      1. 아이콘 = 보석 아이콘 출력
      1. 요구량 = 보석 x 1
    1. 길드코인
      1. 아이콘 = 길드코인 아이콘 출력
      1. 교환량 = 길드코인 x 1
  1. 교환량 정보
    1. 상단 텍스트 = 길드코인 교환 개수
    1. 교환 개수 인디케이터
      • + , - 버튼 : 선택 시 각각 1개 씩 교환 개수 증감
      • 숫자 필드 : 교환할 길드코인 개수 출력
        • 선택 시 수동 입력 필드로 작동
        • 필드가 비어있을 때 기본값 출력

        ※ 숫자만 입력 가능

        ※ 최소 = 1 , 최대 = 1000000 , 기본값 = 1

      • -100 버튼 : 현재 숫자 필드 - 100개
      • +100 버튼 : 현재 숫자 필드 + 100개
      • 최대 버튼 : 선택 시 숫자 필드가 현재 보유한 보석으로 교환할 수 있는 최대치로 변경
  1. 보석 소모량

    출력 형태 : 소모 [보석 아이콘] [소모량 개수]

    ※ 보석 보유량 < 소모량 일 경우 소모량 텍스트 붉은색 출력

  1. 버튼
    1. 취소 = 팝업 닫기
    1. 교환 = 소모량 만큼 보석 → 길드코인 교환

      ※ 보석 보유량 < 소모량 일 때 교환 버튼 선택 시 팝업 닫기 후 보석 교환 팝업 출력.

5-2. 뽑기권 구매 팝업

  • 출력할 뽑기권 정보
    1. 뽑기권 Id 참조 ProductInfo → FixedRewardId 참조
    1. FixedRewardInfo → Item Id로 해당 아이템 정보 호출
  • 뽑기권을 구매할 수 있는 팝업
  • 해당 팝업 출력 시 상단 바가 출력되어야 함
  1. 타이틀 = [선택한 뽑기권] 교환
  1. 가격 정보
    1. 뽑기권 비용 아이콘
      1. 아이콘 = 뽑기권 요구 재화 아이콘 출력
        • 아이콘 = 뽑기권 Id 참조 ProductInfo → Payment 참조
        • 해당 재화 아이콘 출력
      1. 요구량 = 길드코인 x [가격]
        • 가격 = 뽑기권 Id 참조 ProductInfo → Price 참조
    1. [선택한 뽑기권] 아이콘
      1. 아이콘 = [선택한 뽑기권] 아이콘 출력
      1. 교환량 = [선택한 뽑기권]
  1. 구매량 정보
    1. 상단 텍스트 = [선택한 뽑기권 이름] 구매 개수
    1. 구매 개수 인디케이터
      • + , - 버튼 : 선택 시 각각 1개 씩 구매 개수 증감
      • 숫자 필드 : 교환할 길드코인 개수 출력
        • 선택 시 수동 입력 필드로 작동
        • 필드가 비어있을 때 기본값 출력

        ※ 숫자만 입력 가능

        ※ 최소 = 1 , 최대 = 1000000 , 기본값 = 1

      • -10 버튼 : 현재 숫자 필드 값 - 10개
      • +10 버튼 : 현재 숫자 필드 값 + 10개
      • 최대 버튼 : 선택 시 숫자 필드가 현재 보유한 요구 재화로 뽑기권을 구매할 수 있는 최대치로 변경
  1. 요구 재화 소모량

    출력 형태 : 소모 [요구 재화 아이콘] [소모량 개수]

    ※ 요구 재화 보유량 < 소모량 일 경우 소모량 텍스트 붉은색 출력

  1. 버튼
    1. 취소 = 팝업 닫기
    1. 구매 = 숫자 필드에 입력된 개수 만큼 [선택한 뽑기권] 구매

      ※ 요구 재화 보유량 < 소모량 일 때 구매 버튼 선택 시 팝업 닫기 후 해당 재화 교환 팝업 출력.

      ※ 재화 보유량이 부족하여 교환 팝업 출력 시 부족한 재화량이 숫자 필드에 입력되어 있음
